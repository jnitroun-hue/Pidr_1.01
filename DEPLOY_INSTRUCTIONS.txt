=====================================================
ИНСТРУКЦИЯ ПО ДЕПЛОЮ НОВЫХ ТАБЛИЦ P.I.D.R.
=====================================================

ФАЙЛ ДЛЯ ДЕПЛОЯ: NEW_TABLES_DEPLOY.sql

ЧТО СОЗДАЕТСЯ:
1. Таблица _pidr_transactions (транзакции пользователей)
2. Индексы для производительности
3. RLS политики безопасности  
4. Функции update_user_balance() и get_user_balance()
5. Триггеры для updated_at

КАК ДЕПЛОИТЬ:
1. Зайдите в Supabase Dashboard
2. SQL Editor → New Query
3. Скопируйте содержимое NEW_TABLES_DEPLOY.sql
4. Выполните запрос
5. Проверьте сообщения об успешном создании

ПРОВЕРКА:
- В Tables должна появиться _pidr_transactions
- В Functions должны быть update_user_balance, get_user_balance
- Запустите тестовый запрос для проверки

ВАЖНО:
- Файлы .sql НЕ коммитятся в git (добавлены в .gitignore)
- Деплой только через Supabase Dashboard
- После деплоя можно тестировать API /api/bonus и /api/user/balance
